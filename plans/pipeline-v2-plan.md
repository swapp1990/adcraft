# AdCraft Pipeline v2 — Smart Clip Generation with Brand Identity

**Status: DRAFT** (March 1, 2026)

## 1. Problem Statement

Every ad generated by AdCraft v1 gets critiqued for the same fundamental gaps: no brand identity, no product shots, and no call to action. This happens because the pipeline sends pure text prompts to Grok Imagine Video, which cannot reliably render specific logos, text, or UI screens. The critique loop is useless if the pipeline can never address the critique's feedback.

## 2. Solution Overview

Upgrade the pipeline so Gemini acts as a creative director that decides *how* to generate each clip. Some clips (lifestyle, atmosphere) stay as direct text-to-video. Others (brand intro, product shots, CTA end card) use an image-first workflow: generate a composed still frame via Gemini image generation, then optionally animate it with Grok image-to-video. The first and last clips always carry brand identity and CTA respectively, ensuring every ad has the basics that critiques demand.

## 3. Comparison Table

| Aspect | Pipeline v1 | Pipeline v2 |
|--------|------------|-------------|
| Clip generation method | All clips: text → Grok video | Per-clip: agent decides method |
| Brand identity | None — text-to-video can't render logos | Intro clip with brand name/logo via image gen |
| Call to action | None | CTA end card via image gen (readable text) |
| Product visibility | Random — prompt may or may not show product | Product shots via image-first (composed frame) |
| Script structure | Generic scenes | Typed scenes: intro, lifestyle, product, cta |
| Image generation | Not used | Gemini image gen for composed frames |
| Cost per ad | ~5 Grok video calls | ~5 Grok calls + 2-3 image gen calls |
| Time per ad | ~13 min | ~14-15 min (image gen adds ~30s per frame) |

## 4. User Flow

```
1. User enters concept: "ZenFit — minimalist home workout app for busy professionals"
2. Clicks "Generate Ad"
3. Pipeline v2 stages:

   Stage 1 — Script (Gemini)
   Gemini writes a typed script:
   [
     { type: "intro",     method: "image_static",  description: "ZenFit logo on gradient..." },
     { type: "lifestyle", method: "direct_video",   description: "Person stretching at dawn..." },
     { type: "product",   method: "image_animate",  description: "ZenFit app UI showing..." },
     { type: "lifestyle", method: "direct_video",   description: "Quick bodyweight workout..." },
     { type: "cta",       method: "image_static",   description: "Download ZenFit — App Store badge..." }
   ]

   Stage 2 — Clip Prompts (Gemini)
   Each scene gets a prompt optimized for its method:
   - image_static: prompt tuned for still image composition
   - image_animate: prompt tuned for still image + animation direction
   - direct_video: prompt tuned for video generation

   Stage 3 — Generate Clips (parallel, mixed methods)
   - intro (image_static): Gemini image gen → static frame → ffmpeg pad to 4s
   - lifestyle (direct_video): text → Grok Imagine Video (unchanged)
   - product (image_animate): Gemini image gen → Grok image-to-video
   - lifestyle (direct_video): text → Grok Imagine Video (unchanged)
   - cta (image_static): Gemini image gen → static frame → ffmpeg pad to 4s

   Stage 4 — Analyze (unchanged)
   Stage 5 — Assemble (unchanged — already handles mixed clip sources)

4. Video plays in UI with brand intro and CTA end card
5. Critique scores higher because brand identity and CTA are present
```

## 5. Scope

### In Scope (P0 — Must Have)

- **Typed scene structure** — script stage outputs `type` (intro/lifestyle/product/cta) and `method` (image_static/image_animate/direct_video) per scene
- **Gemini image generation** — new `image_gen.py` module using `gemini-3-pro-image-preview` to generate composed still frames
- **Image-to-video via Grok** — extend `xai.py` to accept an image URL/bytes and animate it using Grok's image-to-video endpoint
- **Static frame to clip** — ffmpeg converts a still image to a 3-5s video clip (with optional Ken Burns pan/zoom)
- **Enforced intro and CTA** — first scene is always type "intro", last is always type "cta"; Gemini cannot skip these
- **Updated script prompt** — Gemini prompt explicitly requires brand name in intro, CTA text in last scene
- **Mixed parallel generation** — Stage 3 dispatches each clip to the correct generation method based on `method` field

### Should Have (P1)

- **Ken Burns effect** — ffmpeg slow zoom/pan on static frames to add motion
- **Critique-informed regeneration** — pass critique feedback into the script prompt on regenerate, so Gemini addresses previous weaknesses
- **Method confidence** — Gemini outputs a confidence score per method choice; fallback to direct_video if low confidence
- **User override** — frontend UI to manually set method per clip

### Out of Scope (v2+)

- User-uploaded logos or brand assets
- Custom fonts or brand guidelines input
- Video-to-video style transfer
- Audio/voiceover generation
- Real product photography integration

## 6. Risks & Mitigations

| Risk | Impact | Mitigation |
|------|--------|------------|
| Gemini image gen produces unreadable text in intro/CTA | High | Prompt engineering for simple, large text; fallback to ffmpeg drawtext overlay on solid gradient |
| Image-to-video animation distorts the composed frame | Med | Keep product shots as image_static if animation quality is poor; let analyze stage flag bad clips |
| Added complexity increases generation time significantly | Med | Image gen is fast (~5-10s per image); only 2-3 clips use it; total time increase ~1-2 min |
| Grok image-to-video API may not be available or may differ from text-to-video | High | Check xAI API docs for image-to-video support; fallback to image_static if not available |
| Gemini model ID changes or image gen gets deprecated | Med | Use model lookup from docs per CLAUDE.md rules; abstract model ID to config |

## 7. Success Criteria Checklist

### Script Structure
- [ ] Script output includes `type` field per scene (intro/lifestyle/product/cta)
- [ ] Script output includes `method` field per scene (image_static/image_animate/direct_video)
- [ ] First scene is always type "intro" with brand name in description
- [ ] Last scene is always type "cta" with call-to-action text in description
- [ ] At least one middle scene is type "product" with product/app description

### Image Generation
- [ ] New `image_gen.py` module generates images via Gemini image generation API
- [ ] Generated intro image includes brand name text that is readable
- [ ] Generated CTA image includes action text (e.g., "Download Now") that is readable
- [ ] Images are uploaded to S3 and URL is available for downstream stages

### Clip Generation (Mixed Methods)
- [ ] `image_static` clips: still image → ffmpeg pads to target duration as video
- [ ] `image_animate` clips: still image → Grok image-to-video → video bytes
- [ ] `direct_video` clips: text prompt → Grok text-to-video (unchanged from v1)
- [ ] All three methods produce clips that ffmpeg can assemble (same codec, resolution)

### Pipeline Integration
- [ ] Stage 3 reads `method` field and dispatches to correct generation function
- [ ] All clips generated in parallel regardless of method
- [ ] Failed image_animate gracefully falls back to image_static
- [ ] Final assembled video has intro as first clip and CTA as last clip

### Critique Improvement
- [ ] Ads generated with v2 pipeline score higher on brand identity in critique
- [ ] Critique no longer flags "no brand identity" or "no CTA" as top weakness
- [ ] Score improves from ~3/10 to 5+/10 on repeat generation

### No Regressions
- [ ] Direct text-to-video clips still work (method=direct_video unchanged)
- [ ] Existing generate and critique API endpoints work with same request format
- [ ] Frontend works without changes (pipeline changes are backend-only)
- [ ] Job history shows v2 jobs correctly

## 8. End-to-End Test List

**E2E-1: Script typing**
1. Submit: generate job with concept "ZenFit — workout app"
2. Verify: script output has `type` and `method` fields per scene
3. Verify: first scene type is "intro", last is "cta"
4. Verify: at least one scene has type "product"

**E2E-2: Image generation for intro**
1. Run pipeline with a concept that has a clear brand name
2. Verify: intro clip was generated via image_static method
3. Verify: intro image was uploaded to S3
4. Verify: intro clip in final video shows brand-related imagery

**E2E-3: Image-to-video for product shot**
1. Run pipeline with product scene using image_animate method
2. Verify: still image was generated first
3. Verify: Grok image-to-video was called with the image
4. Verify: resulting clip has motion (not just a static frame)

**E2E-4: Mixed method assembly**
1. Generate a full ad with mixed methods (some image_static, some direct_video)
2. Verify: all clips assemble into a single video without errors
3. Verify: video plays correctly with no codec mismatches

**E2E-5: Critique score improvement**
1. Generate an ad with v2 pipeline
2. Run critique on it
3. Verify: critique does not flag "no brand identity" as top weakness
4. Verify: score is higher than v1 baseline (~3/10)

**E2E-6: Fallback on failure**
1. Simulate image_animate failure (e.g., Grok image-to-video times out)
2. Verify: clip falls back to image_static (still frame as video)
3. Verify: ad still assembles and completes

## 9. Manual Testing Checklist (Post-Deploy)

### Quick Smoke Test (2 min)
- [ ] Submit a generate job with "TestBrand — test product" concept
- [ ] Job completes without errors
- [ ] Final video has a visually distinct first clip (intro) and last clip (CTA)

### Quality Check (5 min)
- [ ] Watch the full generated ad — intro feels like a brand intro, not a random scene
- [ ] CTA end card has some form of action prompt visible
- [ ] Product scene shows something related to the actual product concept
- [ ] Transitions between image-based and video-based clips are smooth

### Critique Validation (3 min)
- [ ] Run critique on the v2-generated ad
- [ ] Score is 5/10 or higher
- [ ] Top weakness is NOT "no brand identity" or "no call to action"
- [ ] Recommendation is about refinement, not fundamental missing elements

### No Regressions (2 min)
- [ ] Frontend still loads and works
- [ ] Old jobs in history still display correctly
- [ ] Critique endpoint works on v2 videos
